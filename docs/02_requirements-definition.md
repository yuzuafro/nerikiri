## 1. システム概要

本システムは、Webブラウザ上で動作するシングルページアプリケーション(SPA)である。画面中央に粘土を模した3Dメッシュを表示し、ユーザーはマウス操作でメッシュ表面を変形(スカルプト)・彩色することで、粘土細工(将来的には練り切りの成形)を疑似体験できる。

サーバーサイド処理は持たず、すべての処理はクライアント(ブラウザ)内で完結する。

## 2. 機能要件

### 2.1 粘土表示機能

| 要件ID | 要件 | 対応要求 |
|---|---|---|
| F-01-01 | 起動時に球形の粘土(細分化された多面体メッシュ)を画面中央に表示する | UR-02 |
| F-01-02 | 粘土は陰影の付いたマットな質感で描画し、形状の凹凸が視認できること | UR-02 |
| F-01-03 | 初期形状として「球」「基本形(上面が平らになりつつ下が少しすぼんだ、練り切りの基本の形)」の2種類を選択できる | UR-09 |
| F-01-04 | 空間の基準として、粘土が載るまな板状の台、正面を示すマーカー、高さの目盛りを表示する。粘土の底面は台の上面に接地し、台に影を落とすことで接地感を得る | UR-12 |

### 2.2 カメラ操作機能

| 要件ID | 要件 | 対応要求 |
|---|---|---|
| F-02-01 | マウス右ドラッグ(または修飾キー+ドラッグ)で粘土の周囲を回転して眺められる | UR-02 |
| F-02-02 | マウスホイールでズームイン・ズームアウトができる | UR-02 |
| F-02-03 | カメラ操作中に誤って粘土が変形されないこと | UR-03 |

### 2.3 スカルプト機能

| 要件ID | 要件 | 対応要求 |
|---|---|---|
| F-03-01 | マウス左ドラッグで、ポインタ位置の粘土表面を変形できる | UR-03 |
| F-03-02 | ブラシ種類「押す」: 表面を法線方向に凹ませる | UR-03 |
| F-03-03 | ブラシ種類「引く」: 表面を法線方向に盛り上げる | UR-03 |
| F-03-04 | ブラシ種類「なめらか」: 表面の凹凸を平均化して滑らかにする | UR-03 |
| F-03-05 | ブラシ種類「つまむ」: ブラシ中心に向かって表面を寄せる(練り切りの「摘み」に相当) | UR-03 |
| F-03-06 | ブラシ種類「ふくらます」: 各頂点をその頂点自身の法線方向に膨張させる | UR-03 |
| F-03-07 | ブラシ種類「ならす」: ブラシ接触面を平面に近づける(ヘラで押さえる操作に相当) | UR-03 |
| F-03-08 | ブラシの半径を 0.05〜0.8(粘土半径1.0 に対する比)の範囲で調整できる | UR-04 |
| F-03-09 | ブラシの強さを 0.1〜1.0 の範囲で調整できる | UR-04 |
| F-03-10 | ブラシの影響は中心で最大、外縁で0となる滑らかな減衰(フォールオフ)を持つ | UR-03 |
| F-03-13 | 変形の効きは練り切りの繊細な手作業に見合う控えめなものとし、ポインティングデバイスのイベント頻度に依存しない(適用頻度の制限を行う) | UR-13 |
| F-03-11 | ポインタが粘土上にあるとき、ブラシの影響範囲をカーソル表示する(三角棒は線幅を表示) | UR-10 |
| F-03-12 | 道具「三角棒」: V字断面の細い線をなぞった軌跡に沿って彫れる。(1)手ぶれ補正により線がジグザグにならない、(2)軌跡は線分として連続的に彫られ点線にならない、(3)1ストローク内では同じ場所を何度なぞっても深さが一定(実物の一筆と同じ)、(4)ストロークを重ねると深くなる、(5)線の太さ・深さは「強さ」のみで決まる(実物の道具は形状固定のため) | UR-14 |

### 2.4 彩色機能

| 要件ID | 要件 | 対応要求 |
|---|---|---|
| F-04-01 | ブラシ種類「塗る」: ブラシ範囲の表面に選択色を塗れる(頂点カラー) | UR-06 |
| F-04-02 | 「全体を塗る」操作で粘土全体を選択色にできる | UR-06 |
| F-04-03 | 練り切りの定番色(白・桃・紅・抹茶・黄・紫・空・小豆)のパレットから色を選択できる | UR-06 |
| F-04-04 | カラーピッカーで任意の色も選択できる | UR-06 |

### 2.5 履歴機能

| 要件ID | 要件 | 対応要求 |
|---|---|---|
| F-05-01 | 「元に戻す」(Undo)で直前のストローク(ドラッグ開始〜終了)単位で状態を戻せる | UR-05 |
| F-05-02 | 「やり直す」(Redo)で戻した操作を再適用できる | UR-05 |
| F-05-03 | 履歴は直近30ステップ以上保持する | UR-05 |
| F-05-04 | キーボードショートカット Ctrl+Z / Ctrl+Y(Ctrl+Shift+Z)に対応する | UR-05 |

### 2.6 リセット・保存機能

| 要件ID | 要件 | 対応要求 |
|---|---|---|
| F-06-01 | 「リセット」で選択中の初期形状・初期色に戻せる(誤操作防止のため確認を行う) | UR-07 |
| F-06-02 | 「保存」で現在の形状・色をJSONファイルとしてダウンロードできる | UR-08 |
| F-06-03 | 「読み込み」で保存したJSONファイルを読み込み、作業を再開できる | UR-08 |
| F-06-04 | 不正なファイルを読み込んだ場合はエラーメッセージを表示し、現在の作業内容を保持する | UR-08 |

|---|---|---|
| F-07-01 | 画面は日本語表記とする | UR-10 |

## 3. 非機能要件

| 要件ID | 分類 | 要件 |
|---|---|---|
| NF-01 | 性能 | 標準初期メッシュ(約1万頂点)のスカルプト操作中も概ね30fps以上、通常時60fpsを目標とする |
| NF-02 | 性能 | 初回ロード(JS+アセット)は 2MB 以下を目標とする |
| NF-03 | 互換性 | Chrome / Edge / Firefox / Safari 最新版(WebGL 2.0 対応環境)で動作する |
| NF-04 | 配布 | ビルド成果物は静的ファイルのみで構成し、任意の静的ホスティングで公開できる |
| NF-05 | セキュリティ | 外部サーバーへのデータ送信を行わない。読み込むファイルはユーザーが明示的に選択したものに限る |
| NF-06 | 保守性 | 変形・履歴等のコアロジックは描画ライブラリ(Three.js)に依存しない純粋なモジュールとして分離し、単体テスト可能とする |
| NF-07 | 拡張性 | ブラシはプラグイン的に追加できる共通インタフェースで実装する(将来要求 FR-01〜03 への布石) |

## 4. スコープ外(本リリースでは実装しない)

- 練り切り専用道具(三角べら等)の再現、包餡・茶巾絞りのシミュレーション(将来要求)
- タッチ・ペンタブレット対応
- メッシュのトポロジー変更(切断・結合・細分化の動的変更)
- 体積保存(粘土の体積が厳密に一定になる物理シミュレーション)
- 画像書き出し・SNS共有

## 5. 用語定義

| 用語 | 定義 |
|---|---|
| スカルプト | メッシュ頂点を移動させて形状を変形する操作 |
| ブラシ | スカルプト・彩色の単位となる道具。半径・強さ・種類を持つ |
| ストローク | マウスボタン押下から離すまでの一連のブラシ適用。Undoの単位 |
| フォールオフ | ブラシ中心からの距離に応じた影響度の減衰関数 |
| 頂点カラー | メッシュ頂点ごとに保持する色情報。面の色は頂点色の補間で決まる |
